Searching JSON documents

In this movie, we'll take a very quick look at how you can index and search JSON documents using RediSearch. In order to have some JSON data that I can index and search, I'm going to set up a few key value pairs using JSON.SET. Here, corresponding to the key emp: 1, I have the JSON object for Gerald. Next, I use JSON.SET once again, this will now set the document for emp: 2 This document contains information about Naziya located in Bangladesh. I'll set up a third document with the emp prefix, emp: 3 is the key and this document is a JSON document for Steffii located in New Zealand. Before we set up an index, I'll use JSON. MGET to confirm that I have the three documents with the emp prefix. Yes indeed I do, Gerald, Naziya, and Steffi. We can now create a search index on our JSON documents using FT .CREATE. The name of the index is empIdx. Notice that I use ON JSON prefix because we are indexing into JSON documents. We need to use the command ON JSON prefix rather than ON HASH prefix that we've seen earlier. ON HASH prefix is for Redis hashes, not JSON. We'll index all documents that has the emp: prefix in the key. And notice how I specify the schema for the JSON documents. The $.Id AS ID numeric will index this field as numeric values. The AS ID specifies that the name ID will be an alias for the field ID, the $.Name AS Name TEXT SORTABLE will allow us to sort based on the name field. The $ .Skills .Asterisk AS SKILLS tag will allow us to search for the nested fields under the skills attribute as tags and the $ .Major AS Major TEXT will treat the major attribute as a text field. Now let's set this index up and we'll see how we can use RediSearch with this index. If you run an FT .INFO command on the empIdx that we just created, the main thing to notice here is the key type. The key type is JSON. Earlier, all of the indices that we created were on Redis hashes. There, the key type was HASH. Once you've set up the index for the JSON documents and you have aliased every JSON field in the index, well, the search queries that we run will be very familiar to you here. I search within the ID field for all documents with the range one, two, both inclusive and this gives me Gerald and Naziya in the result, Gerald with Id 1 and Naziya with Id 2. Next I'll do a wildcard search in the major field at Major: Computer * and I'll return just the name and major fields for the documents that match the search query. And once again, I get Gerald and Naziya as a match. Gerald with a major of computer science and Naziya with a major of computer engineering. The skills attribute in our JSON object has been indexed as a tag. I search within the skills attribute @Skills, and I specify the tags within curly braces, [Redis or Tableau,] and the JSON documents that match this query are the ones for Steffi and Naziya. Notice that their skills array contains either Redis or Tableau.
