
Working with JSON objects

In this demo, we'll see how we can use the RedisJSON module part of a Redis Stack to work with JSON data. Here I am with the Redis command line client open and I'm going to use the JSON.SET command to set the value of a key. The value node is in the JSON format. The key here is user_001. Next, we have the path where we want this value to be set. The path here is just a $ which stands for the root path. That is the default for new keys. We'll explore paths in more detail later. And then we have the JSON structure. The JSON structure contains ID, name, location and whether a user is active. The user_001 key is now assigned a JSON value. We can use the JSON.TYPE command to check the value of the user_001 key and you can see that the value is of type object. This object here refers to a JSON object, that is, the value type for this key. You can check the types of the individual keys in the JSON object as well. Here I want to know the key for the ID within my JSON object and I specify the path to ID, which is just $.Id. And here you can see that the value of the ID is of type integer. If you want to access and check the type of the name field for our user object, you can use $.Name along with the JSON.TYPE command and you can see that the name field has a value type "string." Let's try this once again with the active field. $.Active is the JSON path you'll specify to access the active field and you can see it's of a boolean type. If you want to access the JSON object associated with the key, you can use the JSON.GET command. JSON.GET user_001 will give you the object associated with this key. The object is a JSON value. You can use the JSON path starting with a $ to access the values for individual fields in your JSON object as well. This will give you the value of the ID for user_001, the ID is just one. The fields in the JSON object are case sensitive, so if you use $.id with the lowercase i, you won't get anything in the result. That's because when we had specified the JSON object we had Id with an uppercase I. You can use the JSON.GET command to access multiple fields in an object as well. Here I want to access the ID as well as the name fields for user_001 and this will return both of these bits of information to me in the form of a JSON object. If you want to view your JSON data in the Redis serialization protocol or Resp specification, you can use the JSON.RESP command. So JSON.RESP user 001 will give you the value using the read serialization format. The open curly brace indicates that this is a JSON value. If you want to delete specific fields from a JSON object, that's possible using the JSON.DEL command. Specify the JSON path to the field that I'm deleting, then name property of the JSON object. And now if you perform a JSON.GET on user_001, you can see that the resulting JSON object does not have a name. And of course if you want to delete the entire key value pair that is, get rid of the JSON object altogether, you can do a JSON.DEL and specify the key. This means that this key with the JSON value no longer exists, and you can confirm this by performing a JSON.GET on user_001, you get nil as a return value.
