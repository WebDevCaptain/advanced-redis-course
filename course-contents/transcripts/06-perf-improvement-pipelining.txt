
Performance improvement with pipelining

We've discussed that Redis pipelining is a technique that we use to improve performance by issuing multiple commands at once without waiting for the response to each individual command. Let's now prove that pipelining does indeed improve performance. Here in my current working directory, I have two different files that perform the same set of operations on my Redis database server. In one of the files, run_without_pipeline.py, I'm going to run those operations without using pipelining, but in the second file, run_with_pipeline, I'm going to run the operation as a batch using pipelined commands and we'll see what the difference is in round trip time. Let's first take a look at the Python program where we execute the commands without using a pipeline. I have a function defined here called run_ without_pipelining. Within that function on Line 6, I instantiate a connection to the Redis server on my local machine. I set a new value in my database called without_pipeline and initialize that to zero. I then start a timer on Line 10 and then I run a for loop a 100, thousand times and I increment this without_pipeline key. 100,000 times. This is on Line 13. Because I'm performing the increment operation on the Redis object directly, these operations are performed without pipelining. That is, we send the first increment command to the server, wait for the response, send the second increment command, and so on 100,000 times. Once all of these operations are complete, On Line 15, I print out the time taken to perform these 100,000 operations without pipelining. And on Line 17, I print out the final value of the without_pipeline key, that should be 100,000. I'll now switch back to the terminal window and run my Python code, run_without_ pipeline.py. Remember this performs 100,000 operations without using pipelining. Now you might have to wait for a bit and you can see that it took about 12.62 seconds for these 100,000 operations to be completed. The final value of our without_pipeline counter is 100,000. Now, let's see how we can use pipelining to improve the performance of our code. Now here in this file, run_with_pipeline.py, I have defined a function called run_with_pipelining that allows me to batch the operations that I sent to the Redis server using pipelining. I connect to my Redis server on Line 6, that is, the Redis object and then using the Redis object, I access the pipeline object on Line 7. I then set the width pipeline counter value to be zero. I then start a timer on Line 11 and run a for loop for the same number of iterations, 100,000 iterations, and in every iteration, I increment the width pipeline counter. The 100,000 increment operations are not performed one by one. Instead, they're executed as a pipeline operation when we call pipeobj.execute on Line 16. Once the operation is complete, I print out to screen how much time it took for this pipelined operation. This is on Line 18 and on Line 20, I print out the final value of the width pipeline counter. Back to the terminal window, I'm now going to run this Python code that performs the same increment operation with pipelining. And you'll see this time the operations were executed blazingly fast. They're completed within 1.85 seconds. Pipelining thus significantly improve the performance of our commands.
