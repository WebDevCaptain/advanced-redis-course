Before we dive into the advanced features of Redis, that's going to be the main focus of this course, here is a quick overview of what exactly Redis is all about. Redis is an open source, in-memory data store, which can be used as a database, cache, streaming engine as well as a message broker. Redis isn't a regular relational or No SQL database, instead, it stores data in memory in the form of key value pairs. Any data that you store in Redis is actually stored in memory, which means the lookup for this data is very, very fast. Redis can be thought of as a very versatile in-memory data structure server. Redis can store a variety of different types of data. You can use Redis to store strings, numbers, complex data types such as lists, sets, sorted sets, and even streams. Redis serves a very different use case as compared to traditional databases. Redis is actually used to build infrastructure for real time applications, and these real time applications can involve streaming data, messaging applications. Redis can be used as a cache for frequently accessed data in your website. Redis can perform complex computations and can also be used to maintain session state. A very powerful feature that Redis has is that it's extremely fast and performant. It provides extremely low latency and high throughput for all of its operations. So what are some of the capabilities that Redis has to offer? Redis is highly programmable. You can do server-side scripting with Lua and other server-side stored procedures. Redis is also highly extensible. Redis offers a module API that allows you to build custom extensions using programming languages such as C, C++, and Rust. Redis is primarily used as an in-memory data store. That means your dataset is kept in memory for fast access, but you can also choose to store this data in persistent storage for permanent storage of your data. Redis is highly scalable and can be used with huge amounts of data. Redis offers clustering your data as horizontally scaled with hash-based sharding of your data. Your data is sharded and split across the different nodes of the Redis cluster. And finally, Redis is highly available. Redis offers replication with automatic failover for standalone as well as cluster deployments. Now let's take a look at the topics that I'll be covering in this course. All of these topics assume basic familiarity and understanding of the Redis data store. In this course, we'll focus on how we can optimize our operations using Redis Pipelining. We'll then see how we can run benchmark tests on our Redis server. We'll see how we can perform search operations, including full text search using RediSearch. And finally we'll work with JSON documents using RedisJSON. Redis uses a client server protocol to run its commands. We'll see how you can use Pipelining to optimize roundtrip times to the Redis server by batching your commands together. We'll then see how you can run benchmarking operations on Redis using the Redis benchmark utility. We'll use this to assess the performance of various Redis operations. RediSearch allows us to build indices optimized for search operations on our data stored in Redis. This enables querying, secondary indexing, and full text search for Redis. And finally with the RedisJSON extension, you get JSON support for the Redis store. You can store, update and retrieve JSON values IN Redis. Now RediSearch and RedisJSON are extensions available as a part of Redis Stack. Redis Stack extends the core Redis store to provide new data models, data processing, and debugging tools.
