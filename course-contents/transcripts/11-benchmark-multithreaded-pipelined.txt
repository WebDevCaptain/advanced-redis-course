Now, it's quite possible that you might want to run your benchmark tests in a loop, that is, you want to test to continue till you explicitly stop running the test. And you do this using the -l flag. -l will run the commands that we have specified here, the GET and the SET command in a loop. The -n 1 flag will run just one command at a time and this will go on until you explicitly stop the loop by hitting Ctrl C. Go ahead and hit "Enter" and you'll see that the test will keep running and at some point you can go ahead and hit Ctrl C to stop these tests running. Now, by default, the benchmark utility runs these tests in a single threaded manner. If you want multithreading, you need to explicitly specify the --threads flag.
